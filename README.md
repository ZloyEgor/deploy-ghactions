**Тема:** Разработка статического веб-сайта с использованием MkDocs, React и автоматическим развертыванием через GitHub Actions

## Аннотация

В рамках данной работы был разработан комплексный веб-проект, объединяющий генератор статических сайтов MkDocs с современными фронтенд-технологиями React и Bootstrap. Проект включает в себя создание пользовательской темы на основе шаблонизатора Jinja2, интеграцию интерактивных компонентов и настройку автоматизированного процесса сборки и развертывания посредством GitHub Actions.

## 1. Постановка задачи

Целью работы являлась реализация статического веб-сайта на основе существующего проекта с последующим расширением функциональности и внедрением современных технологий веб-разработки. Основные требования включали:

1. Дублирование базового проекта и создание собственной темы с использованием Jinja2
2. Интеграция Bootstrap и React для создания современного пользовательского интерфейса
3. Разработка пайплайнов CI/CD для автоматизации процессов тестирования, сборки и развертывания
4. Обеспечение валидации HTML, минификации контента и обработки стилей через PostCSS
5. Развертывание результирующего сайта на платформе GitHub Pages

## 2. Архитектура решения

### 2.1 Структура проекта

Проект организован в соответствии с принципами модульной архитектуры:

```
deploy-ghactions/
├── .github/workflows/     # Конфигурация CI/CD пайплайнов
├── src/                   # Исходный код фронтенд-компонентов
│   ├── components/        # React-компоненты
│   ├── styles/           # SCSS-стили
│   └── scripts/          # TypeScript-модули
├── my-static-site/       # MkDocs проект
│   ├── docs/            # Markdown-контент
│   ├── theme/           # Пользовательская Jinja2-тема
│   └── mkdocs.yml       # Конфигурация MkDocs
├── scripts/             # Скрипты автоматизации сборки
└── dist/               # Директория сборки
```

### 2.2 Технологический стек

Проект построен на основе следующих технологий:

**Backend/Генерация контента:**
- MkDocs 1.5+ как основной генератор статических сайтов
- Python 3.10+ для выполнения серверной логики
- Jinja2 для создания пользовательских шаблонов

**Frontend:**
- React 18 для создания интерактивных компонентов
- TypeScript для типизированной разработки
- Bootstrap 5 как CSS-фреймворк
- SCSS для расширенной стилизации

**Инструменты сборки:**
- Vite как основной сборщик фронтенд-ресурсов
- PostCSS для обработки стилей
- HTML-validate для валидации разметки

**DevOps:**
- GitHub Actions для CI/CD
- GitHub Pages для хостинга

## 3. Реализованные компоненты

### 3.1 Пользовательская тема MkDocs

Разработана комплексная тема на основе Jinja2, включающая:

- Базовый шаблон `base.html` с интеграцией Bootstrap и React
- Специализированный шаблон `home.html` для главной страницы
- Адаптивную навигационную систему с поддержкой многоуровневых меню
- Интеграцию системы поиска с поддержкой русского языка
- Переключатель светлой/темной темы

### 3.2 React-компоненты

Реализован набор интерактивных компонентов:

**SearchWidget** - компонент поиска с возможностями:
- Динамической фильтрации контента
- Подсветки найденных фрагментов
- Интеграции с поисковой системой MkDocs

**TableOfContents** - интерактивное оглавление:
- Автоматическое извлечение заголовков из контента
- Плавная прокрутка к выбранным разделам
- Индикация текущего положения на странице

**ThemeToggle** - переключатель тем:
- Сохранение предпочтений пользователя в localStorage
- Плавные переходы между темами
- Синхронизация с системными настройками

**BackToTop** - кнопка возврата к началу страницы:
- Появление при прокрутке
- Плавная анимация перехода

### 3.3 Система стилизации

Создана модульная система стилей на основе SCSS:

- Переопределение переменных Bootstrap для создания уникального дизайна
- Компонентно-ориентированная архитектура стилей
- Поддержка темной и светлой тем
- Адаптивный дизайн для различных устройств

## 7. Техническая документация

### 7.1 Системные требования

- Node.js версии 20.x или выше
- Python версии 3.10 или выше
- Git для управления версиями

### 7.2 Процедура развертывания

```bash
# Установка зависимостей
npm install
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# Сборка проекта
npm run build

# Локальный запуск
npm run serve
```

### 7.3 Конфигурационные файлы

- `mkdocs.yml` - конфигурация генератора статических сайтов
- `vite.config.ts` - настройки сборки фронтенд-ресурсов
- `package.json` - зависимости и скрипты Node.js
- `requirements.txt` - зависимости Python
